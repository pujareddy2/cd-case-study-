# Compiler Design Viva Preparation

## Frequently Asked Faculty Questions
**Q1. What is the difference between a Compiler and an Interpreter?**
A: A compiler translates the entire source code into machine code before execution, whereas an interpreter translates and executes code line-by-line.

**Q2. What is the role of Lexical Analysis?**
A: It reads the source code character by character and groups them into meaningful sequences called Lexemes, producing Tokens.

**Q3. What are the phases of a compiler?**
A: Lexical Analysis, Syntax Analysis, Semantic Analysis, Intermediate Code Gen, Code Optimization, and Target Code Gen.

**Q4. What is the difference between a Lexeme, a Token, and a Pattern?**
A: A Pattern is the rule (regex). A Lexeme is the actual string matched (e.g., `count`). A Token is the classified output (e.g., `ID`).

**Q5. Why do we need an Abstract Syntax Tree if we have a Parse Tree?**
A: A Parse Tree contains all grammatical details including parentheses and intermediate non-terminals. An AST is condensed, containing only operations and operands, which is much easier to traverse.

**Q6. Explain LALR Parsing.**
A: Look-Ahead Left-to-Right Rightmost Derivation. It is a bottom-up parsing technique used by Yacc/Bison that builds the tree by reducing terminals into non-terminals.

**Q7. What is a Symbol Table?**
A: A data structure used to store information about identifiers, such as their name, type, and scope.

**Q8. Why use Hash Tables for Symbol Tables?**
A: Hash tables provide average O(1) lookup time, which is critical since compilers look up variable properties thousands of times during compilation.

**Q9. What is Three Address Code (TAC)?**
A: Intermediate code where each instruction has at most three operands (two sources, one destination) and one operator. Example: `t1 = a + b`.

**Q10. What is Syntax Directed Translation (SDT)?**
A: Evaluating semantic rules alongside grammar productions. In our project, building the AST occurs concurrently with Bison parsing the grammar.

## Quick-Fire Viva Questions (11-50)
11. What is the output of Lex? (Tokens)
12. What is the output of Yacc? (Parse tree/AST or immediate semantic actions)
13. What language is Lex and Yacc written in? (C)
14. What does `%left` do in Bison? (Specifies left-associativity and operator precedence).
15. How does Bison handle Shift/Reduce conflicts? (By defaulting to Shift, or using precedence rules).
16. What is the `yylval` variable? (A global union used to pass semantic values from Lex to Yacc).
17. What is the `yywrap` function? (Called by lex when it reaches EOF. Returning 1 means no more files).
18. What is `yylex()`? (The function generated by Lex that returns the next token).
19. What is `yyparse()`? (The function generated by Yacc that drives the parsing).
20. What is a Regular Expression used for in compilers? (Defining patterns for tokens).
21. Can regular expressions match balanced parentheses? (No, that requires a Context-Free Grammar).
22. What is a Context-Free Grammar (CFG)? (A set of recursive rules used to generate patterns of strings).
23. What is a terminal? (A token/leaf node in the grammar).
24. What is a non-terminal? (A placeholder that can be expanded by grammar rules).
25. How do you resolve `a + b * c` precedence? (By placing `*` lower in the derivation tree or giving it higher precedence in Yacc).
26. What is an activation record? (A stack frame containing local variables and return addresses for function calls).
27. Why do we generate temporary variables in TAC? (To hold intermediate computation results).
28. What is `lex.yy.c`? (The C code generated by flex).
29. What is `parser.tab.c`? (The C code generated by bison).
30. What is the role of `extern FILE *yyin`? (Points the lexer to read from a specific file instead of standard input).
31. What is Constant Folding? (An optimization technique where constant expressions are evaluated at compile time).
32. What is Dead Code Elimination? (Removing code that does not affect program outcomes).
33. Is TAC platform-dependent? (No, it is machine-independent).
34. What data structure did you use for the AST? (A binary tree implemented using C structs).
35. What is `strdup`? (A C function that duplicates a string, allocating memory for it. Used for identifier names).
36. What happens if an undeclared variable is used? (In Semantic Analysis, the Symbol Table lookup returns NULL, triggering an error).
37. How is Scope managed? (By chaining symbol tables or adding a scope identifier/level to the entry).
38. What is a production rule? (A rule in CFG like `expr -> expr + term`).
39. What is bottom-up parsing? (Constructing the parse tree starting from the leaves and moving to the root).
40. What is a handle? (A substring that matches the right side of a production rule during bottom-up parsing).
41. What is error recovery in Yacc? (Using the `error` token to sync the parser state after a syntax error).
42. Why is C used for writing compilers? (It offers low-level memory control, speed, and standard library support).
43. What is the difference between Flex and Lex? (Flex is the Fast Lexical Analyzer, an open-source alternative to Lex).
44. What is Bison? (GNU's version of Yacc - Yet Another Compiler Compiler).
45. How does the scanner ignore whitespace? (By matching `[ \t\n]+` and doing nothing).
46. Can a lexer detect syntax errors? (No, it only detects invalid characters. Syntax is handled by the parser).
47. How did you traverse the AST to generate TAC? (Post-order traversal: Left child, Right child, then Parent).
48. Why Post-order? (Because operands must be evaluated before the operator is applied).
49. What is a basic block? (A sequence of TAC instructions with no branches in or out).
50. What is a directed acyclic graph (DAG) in compilation? (Used to identify common subexpressions within a basic block).

## Troubleshooting Questions
* "Why am I getting a syntax error on valid input?" -> Check if there are invisible carriage returns (`\r`). The lexer must ignore `\r`.
* "Why does the program crash on ID tokens?" -> Ensure `strdup` is used in Lex so the memory persists when Yacc tries to read `yylval.str`.
